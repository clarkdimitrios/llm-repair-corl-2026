# llm-repair-corl-2026
Abstract: Multi-robot mission specifications written in natural language can be mathematically infeasible when submitted to formal planners: a region may be too small for the requested number of agents, a required capability may be absent from the fleet, or concurrent sub-tasks may exceed available resources. Existing approaches either fail silently or report low-level solver errors that operators cannot act on. We present LLM-REPAIR (Robust Explainable Planning via Analytical Iterative Rectification), a framework that bridges formal verification and large language models to produce feasible, intent-preserving mission plans. Given a natural-language mission description, LLM-REPAIR (i) generates a CaTL (Capability Temporal Logic) formula via a grammar-constrained LLM, (ii) encodes it as a Mixed-Integer Linear Program (MILP) and checks feasibility with Gurobi, (iii) extracts an Irreducible Infeasible Subsystem (IIS) and converts it into a structured natural-language conflict explanation, (iv) queries an LLM for ranked repair alternatives filtered by a parametric intent-loss gate, and (v) tunes both generation and repair prompts offline via GEPA, an evolutionary optimizer. The operator confirms or corrects each LLM inference, ensuring zero silent automation. We evaluate on five infeasibility scenarios spanning capability gaps, capacity violations, and temporal impossibilities. Results show that IIS-grounded diagnosis is the primary driver of repair quality, enabling the system to propose structurally sound repairs within one to two rounds across all scenarios.
